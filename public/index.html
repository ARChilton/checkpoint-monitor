Making events safer

Receiving updates from all checkpoints simultaneously in real-time means you know which teams were last seen when and where, or more importantly which teams haven't been seen for some time!

Enjoy monitoring the event

Designed for use on any mobile device, using Checkpoint Live means you can be on the move and still monitor the event.

Need to be flexible during the event? Add new checkpoints for transports or ad-hoc locations and everyone connected to the event will be updated.

Improving communication

Gone are the days of mishearing via radio or misinterpreting handwriting. Inputs are made by individuals at the checkpoint and distributed to everyone in a readable format, in real-time.

With a chat-room included for every event you can contact all of your checkpoint marshals and they can contact you, clearly and transparently.

What is Checkpoint Live?

Checkpoint Live was developed initially for the Scout and Guide communities to enable real time updates on where the teams of young people are as they participate in orienteering activities. Now it can be used by any event with manned checkpoints/bases.

Checkpoint Live can be used via the website to record when, where and who visited a specific checkpoint/base. With the added ability to record any scores taken at that location.

This replaces the need for radio updates or sending messengers to collect records of team sightings from across the countryside.

Importantly for the organisers it removes the data entry into the HQ system as all logs are entered by the checkpoints. This data can even be copied into Microsoft Excel™

What if I have bad mobile reception?

Being out in the county side means there is no guarantee of mobile reception. While this situation is improving, Checkpoint Live has taken an 'offline first' approach.

Offline first means that this website will continue to work even when your mobile internet signal is intermittent. It does this by storing the recorded base logs to your mobile device before synchronising them with other event users when the mobile device re-connects to the internet.

Password protect base entry

This will require those using the app/website to enter a unique passcode to gain access to their base, this limits user's access only to bases they have the passcode for. If this option is turned off all users with the event log-in will be able to select the base they are logging visits for from a drop down list. (The Admin area always requires a passcode).

Log off route entries

This will allow your event users to log when they see a team, even if they are not due to participate on that base yet. This is useful from a safety perspective as every team sighting is recorded.

Automate time entry

Turning this on will remove the need for users to enter time information into the app/website when recording a log entry. Instead the time in and out of the base is set as to the time that the submit log button is pressed. This is useful for quicker submission of logs.

Live feed to a tracking server

Turning this on will enable a live feed of locations to be sent to a tracking server whenever a log is recorded; this remains at the user's discretion to enable location services on their device and only applies to checkpoints which have 'record location with log' turned on below.
